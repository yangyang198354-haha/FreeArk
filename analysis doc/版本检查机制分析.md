# 版本检查机制分析

## 1. 服务器URL

`DownloadUtil.checkUpdate`方法通过`HttpHelper.getApkUpdate`发起版本检查请求，使用的服务器URL为：

```
POST /apk-version/check
```

该URL定义在`NetApi.java`接口文件中，是相对路径，实际请求时会拼接服务器基础URL。

### 1.1 服务器基础URL配置
服务器基础URL定义在`HttpConsts.java`文件中，通过`getBaseUrl()`方法获取：

```java
public final String getBaseUrl() {
    return "http://" + HTTP_SERVER_HOST + ":10013/homeauto-contact-screen/contact-screen/screen/";
}
```

其中`HTTP_SERVER_HOST`根据`use_Host_Online`配置动态决定：
- 当`getUse_Host_Online()`为`true`时，使用**正式环境**：`47.117.41.184`
- 当`getUse_Host_Online()`为`false`时，使用**测试环境**：`47.117.47.6`

### 1.2 完整请求URL
结合相对路径，完整的版本检查URL为：
- 正式环境：`http://47.117.41.184:10013/homeauto-contact-screen/contact-screen/screen/apk-version/check`
- 测试环境：`http://47.117.47.6:10013/homeauto-contact-screen/contact-screen/screen/apk-version/check`

## 2. 请求格式

### 2.1 请求头
- `screenMac`: 设备标识（字符串类型）

### 2.2 请求体
请求体为`NetRequest<String>`对象，包含当前应用版本号：

```json
{
  "request": "当前应用版本号"
}
```

其中`NetRequest`类结构如下：
- `request`: T类型（此处T为String，表示当前应用版本号）

### 2.3 请求示例
```
POST /apk-version/check HTTP/1.1
Content-Type: application/json
Accept: application/json
User-Agent: okhttp/4.9.3

{
  "request": "4.0.0"
}
```

## 3. 返回格式

返回数据为`NetResponse<VersionBean>`对象：

### 3.1 NetResponse结构
- `isSuccess`: 布尔值，表示请求是否成功
- `code`: 整数，响应码（0表示成功）
- `message`: 字符串，响应消息
- `data`: VersionBean对象，包含版本更新信息

### 3.2 VersionBean结构
- `version`: 字符串，新版本号
- `url`: 字符串，升级包下载地址
- `updateFlag`: 布尔值，是否需要更新
- `upgradeType`: 整数，升级类型（1普通升级，2静默升级等）
- `description`: 字符串，更新描述

### 3.3 返回示例
```json
{
  "isSuccess": true,
  "code": 0,
  "message": "成功",
  "data": {
    "version": "4.1.0",
    "url": "http://example.com/updates/freeark-4.1.0.apk",
    "updateFlag": true,
    "upgradeType": 1,
    "description": "1. 修复了已知bug\n2. 新增了新功能\n3. 优化了性能"
  }
}
```

## 4. 版本检查流程

1. `DownloadUtil.checkUpdate`方法接收服务器返回的版本号字符串
2. 解析版本号并与本地版本号进行比较
3. 返回比较结果（0表示相同，负数表示本地版本更低，正数表示本地版本更高）

## 5. 升级类型说明

- `upgradeType = 1`: 普通升级，需要用户确认
- `upgradeType = 2`: 静默升级（当无障碍服务启用时自动安装）
