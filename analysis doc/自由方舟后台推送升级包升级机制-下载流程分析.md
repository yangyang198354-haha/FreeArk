# 自由方舟后台推送升级包升级机制-下载流程分析

## 一、升级包来源分析

### 1. 升级包地址获取流程
升级包的下载地址是从服务器通过HTTP接口获取的，具体流程如下：

1. **服务器请求**：设备通过`HttpHelper.getApkUpdate()`方法向服务器发送请求，获取最新版本信息
2. **响应数据结构**：服务器返回`NetResponse<VersionBean>`对象，其中`VersionBean`包含升级包的下载地址
3. **核心字段**：`VersionBean.url`字段存储了升级包的完整下载URL

### 2. VersionBean数据结构
```java
public final class VersionBean {
    private final String version;       // 版本号
    private final String url;           // 升级包下载地址
    private final boolean updateFlag;   // 是否需要更新
    private final int upgradeType;      // 升级类型（0：普通升级，1：强制升级，2：静默升级）
    private final String description;   // 更新描述
    // ... 其他方法
}
```

## 二、升级包下载流程

### 1. 下载触发条件
当满足以下条件之一时，触发升级包下载：
1. `checkVersionBean()`方法返回`true`（新版本号高于当前版本）
2. 系统开启了OTA测试模式（`SystemConfig.INSTANCE.getInstance().getOtaTest()`）

### 2. 下载方法调用链
```
HttpHelper.getApkUpdate() → VersionBean → DownloadUtil.downloadFileByOkHttp(url, forceInstall)
```

### 3. 下载实现细节
在`DownloadUtil.downloadFileByOkHttp()`方法中，实现了完整的下载逻辑：

```java
public final void downloadFileByOkHttp(final String url, final boolean forceInstall) {
    // 1. 创建OkHttp请求
    Call callNewCall = new OkHttpClient().newCall(
        new Request.Builder().url(url).build()
    );
    
    // 2. 异步执行下载
    callNewCall.enqueue(new Callback() {
        @Override
        public void onFailure(Call call, IOException e) {
            // 下载失败处理
        }
        
        @Override
        public void onResponse(Call call, Response response) {
            // 3. 处理响应并保存文件
            String savePath = PathUtils.getExternalAppDataPath() + File.separator + "files";
            FileUtils.createOrExistsDir(savePath);
            
            // 4. 获取文件格式并构建保存文件名
            String fileFormat = LandLeafUtils.INSTANCE.getFileFormat(url);
            String saveFilePath = savePath + File.separator + "1." + fileFormat;
            
            // 5. 创建文件输出流并写入数据
            FileOutputStream fileOutputStream = new FileOutputStream(saveFilePath);
            byte[] buffer = new byte[2048];
            int bytesRead;
            while ((bytesRead = inputStream.read(buffer)) != -1) {
                fileOutputStream.write(buffer, 0, bytesRead);
                // 可选：记录下载进度
            }
            
            // 6. 下载完成后处理
            fileOutputStream.flush();
            // ... 安装逻辑
        }
    });
}
```

## 三、下载关键参数

### 1. 下载地址（url）
- **来源**：服务器返回的`VersionBean.url`字段
- **格式**：完整的HTTP/HTTPS URL（如：`http://example.com/update/upgrade.apk`）

### 2. 强制安装标志（forceInstall）
- **来源**：`VersionBean.upgradeType`字段
- **逻辑**：当`upgradeType == 2`时，`forceInstall = true`
- **作用**：控制下载完成后是否自动安装

### 3. 保存路径
- **路径**：`/storage/emulated/0/Android/data/{应用包名}/files/files/1.{文件格式}`
- **文件命名**：固定使用`1.`前缀，后缀为服务器返回的文件格式（通常为`apk`）

## 四、下载流程总结

1. **触发下载**：通过MQTT接收到升级指令或定时检查更新时
2. **获取地址**：调用HTTP接口获取`VersionBean`，包含升级包下载地址
3. **执行下载**：使用OkHttp异步下载升级包到指定路径
4. **保存文件**：将升级包保存到应用的外部数据目录
5. **下载完成**：下载完成后准备安装（根据`forceInstall`标志决定是否自动安装）

## 五、代码优化建议

### 1. 下载重试机制
当前实现没有下载失败重试机制，建议添加：
```java
private int retryCount = 0;
private static final int MAX_RETRY_COUNT = 3;

// 在onFailure中添加重试逻辑
@Override
public void onFailure(Call call, IOException e) {
    if (retryCount < MAX_RETRY_COUNT) {
        retryCount++;
        LandLeafLog.d$default(LandLeafLog.INSTANCE, "下载失败，重试第" + retryCount + "次", false, false, 6, null);
        // 重新执行下载
        downloadFileByOkHttp(url, forceInstall);
    } else {
        LandLeafLog.e$default(LandLeafLog.INSTANCE, "下载失败，已达最大重试次数", false, 2, null);
    }
}
```

### 2. 断点续传支持
对于大文件下载，建议添加断点续传功能，减少用户等待时间和流量消耗。

### 3. 下载进度回调
当前实现仅在日志中记录下载进度，建议添加回调接口，支持UI显示下载进度。
