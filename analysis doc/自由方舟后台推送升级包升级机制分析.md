# 自由方舟后台推送升级包升级机制分析

## 1. 概述
自由方舟应用支持通过后台（47.117.41.184）推送升级包进行升级，采用MQTT消息推送+HTTP下载的方式实现远程升级功能。

## 2. 升级机制架构

```
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│  后台服务器     │      │  MQTT消息队列    │      │  自由方舟应用    │
│  (47.117.41.184)│─────>│  (CONFIG_APK_UPDATE) │─────>│                │
└─────────────────┘      └─────────────────┘      └─────────────────┘

                                                    └─────────────────┘
```

## 3. 工作流程

### 3.1 后台推送升级指令
1. 后台服务器通过MQTT协议向设备推送升级指令
2. 指令类型为`CONFIG_APK_UPDATE`（定义在MqttConst.java中）

### 3.2 应用接收升级指令
1. `MqttMsgHandler`接收MQTT消息
2. 通过`HardwareCtrlImpl.INSTANCE.getINSTANCE().apkUpdate()`触发升级流程
3. `HardwareCtrlImpl.apkUpdate()`方法启动协程执行升级操作

### 3.3 获取升级信息
1. `HttpHelper.getApkUpdate()`方法向服务器发送升级检查请求
2. 请求通过`NetApi.apkUpdate()`访问`apk-version/check`接口
3. 请求包含：
   - 请求头：screenMac（设备唯一标识）
   - 请求体：当前应用版本名（使用NetRequest<String>包装）

### 3.4 版本比较
1. 服务器返回`VersionBean`对象，包含升级包信息
2. `HttpHelper.checkVersionBean()`检查版本信息完整性和更新标志
3. `DownloadUtil.checkUpdate()`比较当前版本和服务器版本号：
   - 将版本号按点分割（如"1.2.3" → ["1", "2", "3"]）
   - 逐位比较数字大小，决定是否需要升级

### 3.5 下载升级包
1. 如需要升级，调用`DownloadUtil.downloadFileByOkHttp()`下载升级包
2. 使用OkHttp进行下载，支持断点续传
3. 下载进度实时记录并通过日志输出
4. 下载文件保存在`/sdcard/Android/data/包名/files/1.apk`路径

### 3.6 安装升级包
1. 下载完成后，调用`DownloadUtil.installApp()`进行安装
2. 使用Android的DownloadManager进行安装
3. 支持强制安装选项（upgradeType == 2）

## 5. 技术要点

### 5.1 异步处理
- 使用Kotlin协程+Flow处理异步请求
- 所有网络操作在IO线程执行，避免阻塞主线程

### 5.2 错误处理
- 完善的异常捕获机制
- 下载失败时记录详细日志
- 版本格式错误时的容错处理

### 5.3 安全性
- 设备唯一标识screenMac用于身份验证
- 升级包下载链接安全检查
- 支持强制安装选项，确保关键更新能被安装

### 5.4 用户体验
- 下载进度实时显示
- 下载完成后自动安装
- 支持后台下载，不影响用户正常使用

## 6. 升级包信息

`VersionBean`包含以下升级包信息：
- `version`：版本号
- `url`：升级包下载链接
- `updateFlag`：更新标志
- `upgradeType`：升级类型（1-普通升级，2-强制升级）
- `description`：升级描述

## 7. 接口说明

### 7.1 APK升级检查接口
- **URL**：`/apk-version/check`
- **方法**：POST
- **请求头**：`screenMac`
- **请求体**：`NetRequest<String>`（包含当前版本名）
- **响应**：`Flow<NetResponse<VersionBean>>`

## 8. 总结

自由方舟应用的后台推送升级包升级机制采用了MQTT消息推送+HTTP下载的方式，实现了远程、实时的应用升级功能。该机制具有以下特点：

1. **实时性**：通过MQTT消息推送，实现后台实时触发升级
2. **可靠性**：使用OkHttp进行下载，支持断点续传
3. **安全性**：设备唯一标识验证，支持强制安装
4. **用户友好**：下载进度显示，自动安装
5. **可扩展性**：支持不同升级类型和设备类型

该升级机制满足了智能设备远程管理的需求，确保设备始终运行在最新版本，提高了应用的安全性和稳定性。
