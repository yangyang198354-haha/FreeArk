# 自由方舟后台推送升级包升级机制-安装流程分析

## 一、下载完成后是否马上安装的判断条件

下载完成后是否马上安装，取决于**两个关键条件**：

### 1. 升级类型（upgradeType）
从`HttpHelper$getApkUpdate$2$2$1$1.java`中可以看到：

```java
DownloadUtil.INSTANCE.downloadFileByOkHttp(url, this.$it.getUpgradeType() == 2);
```

只有当`upgradeType == 2`时，`forceInstall`参数才会被设置为`true`。

### 2. 无障碍服务是否启用
从`DownloadUtil.java`的安装逻辑中可以看到：

```java
if (AccessibilityHelper.INSTANCE.getInstance().checkAccessibilityEnabled(ProjectConfig.Auto_InstallService_Name) && this.$forceInstall) {
    DownloadUtil.INSTANCE.goInstallActivity(this.$filePath, this.$context);
} else {
    // 显示更新提示对话框
}
```

## 二、安装流程分类

### 1. 自动安装（静默升级）
当满足以下条件时，下载完成后**会马上自动安装**：
- `upgradeType == 2`（静默升级模式）
- 系统已启用相应的无障碍服务

安装实现：直接调用`goInstallActivity()`方法启动系统安装界面

### 2. 手动确认安装
当以下任一条件不满足时，下载完成后**不会马上安装**，而是显示提示对话框：
- `upgradeType != 2`（普通升级或强制升级）
- 无障碍服务未启用

安装实现：显示`UpdateNoticeDialog`对话框，等待用户点击确认后调用`goInstallActivity()`方法启动系统安装界面

## 三、升级类型说明

根据代码分析，系统支持三种升级类型：

| 升级类型值 | 类型名称 | 安装方式 |
|---------|-------|-------|
| 0       | 普通升级 | 手动确认安装 |
| 1       | 强制升级 | 手动确认安装 |
| 2       | 静默升级 | 自动安装（需无障碍服务支持） |

## 四、安装实现细节

### 1. 安装方法（goInstallActivity）
```java
private final void goInstallActivity(String filePath, Context context) {
    File file = new File(filePath);
    Intent intent = new Intent();
    intent.setAction("android.intent.action.VIEW");
    intent.addCategory("android.intent.category.DEFAULT");
    intent.setFlags(BasePopupFlag.OVERLAY_MASK);
    
    // 根据Android版本处理文件URI
    if (Build.VERSION.SDK_INT >= 24) {
        Uri uriForFile = FixedFileProvider.getUriForFile(
            context, 
            context.getApplicationContext().getPackageName() + ".authority", 
            file
        );
        intent.addFlags(1);
        intent.setDataAndType(uriForFile, "application/vnd.android.package-archive");
    } else {
        intent.setDataAndType(Uri.fromFile(file), "application/vnd.android.package-archive");
    }
    
    context.startActivity(intent);
}
```

## 五、关键结论

1. **默认情况**：下载完成后**不会马上安装**，而是显示更新提示对话框，等待用户确认

2. **自动安装条件**：
   - 升级类型为静默升级（`upgradeType == 2`）
   - 系统已启用无障碍服务

3. **安装流程**：
   - 下载完成 → 判断升级类型和无障碍服务状态 → 自动安装或显示提示对话框 → 用户确认 → 启动系统安装界面

4. **特殊情况**：OTA测试模式下，无论版本号如何，都会使用shell命令自动安装
