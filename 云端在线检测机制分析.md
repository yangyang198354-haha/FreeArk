```
                                         订阅主题 +                 触发重连机制
                                         启动心跳检测                    │
                                              │                           │
                                              └─────────────┬─────────────┘
                                                            ▼

```

**关键代码位置**：`MqttClient.java:247-287`

### 2. 连接状态监控流程

| 状态 | 检测方式 | 触发条件 | 处理策略 |
|------|---------|---------|---------|
| **连接成功** | `connectComplete()`回调 | MQTT Broker确认连接 | 订阅主题、初始化HTTP请求、启动设备状态上报 |
| **连接断开** | `connectionLost()`回调 | 网络异常/Broker断开 | 等待2秒后自动重连 |
| **连接失败** | `onFailure()`回调 | 连接超时/认证失败 | 延迟5秒后重试连接 |
| **心跳超时** | MQTT协议层 | 超过KeepAliveInterval未通信 | 自动触发重连 |

## 三、MQTT订阅机制

### 1. 订阅Topic

**唯一订阅Topic**：

```
/screen/service/cloud/to/screen/{mac}
```

**订阅时机**：
- 首次连接成功时订阅
- 重连成功时重新订阅

**代码实现**：
```java
public final void subscribeTopic(final String mac2) {
    mqttAsyncClient.subscribe(
        MqttConst.COMMON_CLOUD_2_SCREEN_TOPIC + mac2,
        0,  // QoS = 0
        null,
        new IMqttActionListener() {
            @Override
            public void onSuccess(IMqttToken asyncActionToken) {
                LandLeafLog.d("mqtt--> 订阅成功 topic:/screen/service/cloud/to/screen/" + mac2);
            }
        }
    );
}
```

### 2. 所有Topic定义

| 常量名 | Topic路径 | 方向 | 用途 |
|--------|----------|------|------|
| `COMMON_CLOUD_2_SCREEN_TOPIC` | `/screen/service/cloud/to/screen/` | 云端→App | **唯一订阅Topic** |
| `COMMON_SCREEN_2_CLOUD_TOPIC` | `/screen/service/screen/to/cloud/` | App→云端 | 响应消息 |
| `COMMON_UPDATA_TOPIC` | `/screen/upload/screen/to/cloud/` | App→云端 | 设备状态更新 |
| `COMMON_UPLOAD_OPT_TOPIC` | `/screen/event/status/change/` | App→云端 | 操作事件上报 |

## 四、消息格式详解

### 1. 完整消息结构

```json
{
  "header": {
    "messageId": "1704067200001",
    "screenMac": "00:11:22:33:44:55",
    "name": "DeviceStatusUpdate",
    "ackCode": 0
  },
  "payload": {
    "code": 200,
    "message": "success",
    "data": {
      "deviceSn": 12345,
      "productCode": 1001,
      "items": [
        {
          "attrTag": 1,
          "attrValue": "25.5"
        }
      ]
    }
  }
}
```

### 2. 消息头（MqttHeaderModel）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `messageId` | String | 是 | 消息唯一标识 |
| `screenMac` | String | 是 | 设备MAC地址 |
| `name` | String | 是 | 消息类型/方法名 |
| `ackCode` | int | 是 | 0=需要回复，1=无需回复 |

### 3. 消息体（MqttPayloadModel）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `code` | int | 是 | 响应码：200=成功，500=失败 |
| `message` | String | 否 | 响应消息描述 |
| `data` | MqttDataModel | 是 | 数据内容对象 |

### 4. 数据模型（MqttDataModel）

| 字段 | 类型 | 适用消息类型 | 说明 |
|------|------|-------------|------|
| `deviceSn` | Long | 设备相关消息 | 设备序列号 |
| `productCode` | int | 设备相关消息 | 产品编码 |
| `items` | List\<T\> | 设备状态/控制 | 设备属性列表 |
| `sceneId` | Long | 场景相关消息 | 场景ID |
| `updateType` | String | 配置更新 | 更新类型 |
| `notifyType` | String | 屏幕通知 | 通知类型 |
| `notifyMsg` | String/Object | 屏幕通知 | 通知内容 |

## 五、心跳机制详解

### 1. MQTT协议层心跳

**配置参数**：

| 参数 | 值 | 说明 |
|------|-----|------|
| `KeepAliveInterval` | 20秒 | 心跳间隔 |
| `ConnectionTimeout` | 3秒 | 连接超时 |
| `AutomaticReconnect` | true | 自动重连 |

**心跳流程**：
```
App ────PINGREQ────> Broker (每20秒)
App <───PINGRESP──── Broker (确认存活)
```

### 2. 应用层心跳（设备状态上报）

**定时周期**：每10分钟上报所有设备状态

**发送Topic**：`/screen/upload/screen/to/cloud/{mac}`

**消息示例**：
```json
{
  "header": {
    "messageId": "1704067800001",
    "screenMac": "00:11:22:33:44:55",
    "name": "DeviceStatusUpdate",
    "ackCode": 1
  },
  "payload": {
    "code": 200,
    "message": null,
    "data": {
      "deviceSn": 12345,
      "productCode": 1001,
      "items": [
        {"attrTag": 1, "attrValue": "25.5"},
        {"attrTag": 2, "attrValue": "60"},
        {"attrTag": 3, "attrValue": "1"}
      ]
    }
  }
}
```

### 3. 重连机制

```
连接失败/断开
       │
       ▼
 等待延迟时间 (5000ms)
       │
       ▼
 重新调用connect()
       │
       ▼
 成功 ←─────┐
       │    │
       │ 失败
       │    │
       ▼    └──→ 循环重试
 恢复正常
```

## 八、在线检测机制总结

### 检测方式

1. **协议层心跳**：MQTT KeepAlive=20秒，通过PINGREQ/PINGRESP自动维护
2. **应用层心跳**：每10分钟主动上报所有设备状态到云端
3. **连接状态监控**：通过`MqttCallbackExtended`实时监听连接状态变化

### 技术特点

- **低功耗设计**：10分钟间隔上报，避免频繁通信
- **异步处理**：基于Kotlin协程实现非阻塞通信
- **可靠传输**：MQTT QoS=0保证消息尽力传递
- **断线重连**：多层重连策略确保连接稳定性
- **设备隔离**：每个设备上报间隔100ms，避免消息拥塞
