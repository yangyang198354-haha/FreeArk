# BUG-FM-004 RCA — 房号筛选段数不匹配

**BUG 编号**：BUG-FM-004
**发现日期**：2026-05-28
**模块**：故障管理 API（`views_fault.py`）
**严重级别**：High（核心筛选功能完全失效）
**状态**：已修复（v0.6.2-FM）

---

## 1. 现象

用户在故障管理界面选择特定房号（如"9 栋 1 单元 604 室"）进行筛选，结果列表返回空，
无法筛出任何故障记录，但生产数据库中确实存在该房号的故障数据。

---

## 2. 根因分析

### 2.1 数据格式不一致（前端 vs 生产 DB）

前端 `building_data.js` 使用 **3 段格式**：

```
栋-单元-房号
示例：9-1-604
```

生产 DB `fault_event.specific_part` 字段存储的是 **4 段格式**：

```
栋-单元-楼层-房号
示例：9-1-6-604
```

生产 DB 实证（`fault_event` 表，2026-05 实测）：

| specific_part | fault_code | product_code | device_sn | cnt |
|---------------|------------|--------------|-----------|-----|
| 9-1-6-604     | error_140  | 270001       | 22154     | 1   |
| 9-1-6-604     | error_194  | 130004       | 22155     | 1   |
| 9-1-6-604     | error_265  | 100007       | 22157     | 1   |

### 2.2 旧过滤逻辑：纯子串匹配

`views_fault.py:81`（旧代码）：

```python
qs = qs.filter(specific_part__icontains=sp)
```

`icontains` 等价 `LIKE '%9-1-604%'`。

字符串 `"9-1-604"` **不是** `"9-1-6-604"` 的连续子串：

```
9-1-6-604
     ^
     此处：字符 '6'，而非 '6', '0', '4'
"9-1-604" 要求 '9-1-' 后面直接跟 '604'，但实际上跟着 '6-604'
```

因此 `LIKE '%9-1-604%'` 对所有 4 段格式 specific_part 一律不命中，查询返回空。

---

## 3. 修复方案

**方案 B（最小化改动，前端零修改）**：后端解析输入格式，对 3 段格式做前缀+后缀组合匹配。

修改文件：`FreeArkWeb/backend/freearkweb/api/views_fault.py`

```python
parts = sp.split('-')
if len(parts) == 3:
    # 3 段格式：栋-单元-房号（如 "9-1-604"）
    prefix = f"{parts[0]}-{parts[1]}-"   # "9-1-"
    suffix = f"-{parts[2]}"               # "-604"
    qs = qs.filter(
        specific_part__startswith=prefix,
        specific_part__endswith=suffix,
    )
else:
    # 4 段或其他格式：icontains 保持向后兼容
    qs = qs.filter(specific_part__icontains=sp)
```

### 边界分析：前缀误匹配风险

问题假设：`startswith('9-1-')` 是否会误匹配 `9-10-...`？

结论：**不会**。
- `'9-10-604'` 的前4个字节为 `'9', '-', '1', '0'`
- `startswith('9-1-')` 要求前4个字节为 `'9', '-', '1', '-'`
- `'0' != '-'`，因此不匹配

`startswith` 是字节级精确前缀匹配，无歧义。

### 多楼层同房号

条件 `startswith('9-1-') AND endswith('-604')` 会同时命中：
- `9-1-5-604`（5楼604）
- `9-1-6-604`（6楼604）
- `9-1-10-604`（10楼604）

这是**预期行为**——用户选择"9-1-604"时，语义是"9栋1单元604室"，
不区分楼层，应返回所有楼层的604室故障记录。

---

## 4. 影响评估

- **修改范围**：`views_fault.py` 仅修改 `specific_part` 过滤分支，不影响其他过滤逻辑
- **前端无修改**：前端继续传递 3 段格式，后端自动适配
- **向后兼容**：4 段格式输入（如 `9-1-6-604`）走 `icontains` 分支，行为不变
- **SQL 注入安全**：`startswith`/`endswith` 均通过 Django ORM 参数绑定，无原生 SQL 拼接
