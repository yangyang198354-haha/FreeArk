## US-01 业主一览座舱全局健康与设备状态

**作为** 业主
**我希望** 一进入舰桥页面就能看到我绑定座舱的整体健康摘要、子系统隔舱状态和各房间设备故障/预警情况
**以便** 无需深入任何子页面就能判断家中哪些设备出了异常

**关联需求**：REQ-FUNC-001、REQ-FUNC-002、REQ-FUNC-003、REQ-FUNC-005、REQ-FUNC-008、REQ-FUNC-009

**优先级**：Must Have

**故事点**：待开发团队评估

**验收标准：**

- **AC-01-01** 整体健康——告警
  - Given 当前座舱存在至少 1 条活跃故障（severity=error，任意子系统或房间）
  - When 业主进入舰桥页面
  - Then 页面顶部健康指示器显示红色"告警"状态（红色菱形 LED 发光 + "告警"文字，色值 `#ff315d`，含 `box-shadow` 红光）

- **AC-01-02** 整体健康——预警
  - Given 当前座舱无活跃故障，但存在至少 1 条活跃预警（severity=warning 或结露预警）
  - When 业主进入舰桥页面
  - Then 健康指示器显示黄色"预警"状态（黄色菱形 LED 发光 + "预警"文字，色值 `#ffd400`）

- **AC-01-03** 整体健康——正常
  - Given 当前座舱无任何活跃故障且无任何预警
  - When 业主进入舰桥页面
  - Then 健康指示器显示绿色"正常"状态（绿色菱形 LED 发光 + "正常"文字，色值 `#27f5b5`）

- **AC-01-04** 加载中
  - Given 舰桥页面正在加载数据（首次请求未返回）
  - When 业主等待加载
  - Then 健康指示器显示灰色"同步中"状态 + 扫描线动画效果

- **AC-01-05** 子系统隔舱——故障
  - Given 新风机组存在 3 条活跃故障
  - When 业主查看舰桥页面子系统 dock 区
  - Then "新风"隔舱以红色/橙红边框显示，标注故障计数"3"，显示损伤闪烁动画

- **AC-01-06** 子系统隔舱——正常
  - Given 水力模块无任何故障或预警
  - When 业主查看舰桥页面
  - Then "水力模块"隔舱以青蓝色霓虹线显示，状态点为绿色，不显示故障/预警计数

- **AC-01-07** 房间隔舱——故障
  - Given "主卧"房间内的设备存在 1 条活跃故障（severity=error）
  - When 业主查看舰桥页面房间网格区
  - Then "主卧"房间隔舱以红色/橙红闪烁线显示，标注故障指示，不显示温度数值

- **AC-01-08** 房间隔舱——正常
  - Given "书房"房间所有设备正常
  - When 业主查看舰桥页面
  - Then "书房"房间隔舱以青蓝色霓虹线显示，状态点为绿色，无故障/预警标记

- **AC-01-09** PLC 独立指示器
  - Given PLC 部分离线
  - When 业主查看舰桥页面
  - Then PLC 在线状态独立指示器显示黄色"预警"（PM 确认：I-04，独立指示器）

- **AC-01-10** 结露预警独立计数角标
  - Given 当前座舱存在 2 条活跃结露预警
  - When 业主查看舰桥页面
  - Then 在页面醒目位置（健康指示器旁）展示结露预警独立计数角标（黄色发光 Badge + "2"），与故障计数分列显示（PM 确认：I-03 方案 B）

- **AC-01-11** 禁止运行参数
  - Given 房间隔舱展示
  - Then 隔舱内不出现温度（°C）、湿度（%RH）、能耗（kWh）等运行参数数值（约束 C-02）

## US-02 战舰透视图的沉浸式视觉呈现

**作为** 业主
**我希望** 舰桥页面呈现为一艘 2D 赛博朋克风格太空战舰的 X 射线透视蓝图
**以便** 获得符合"方舟座舱"主题的沉浸式监控体验

**关联需求**：REQ-FUNC-004、REQ-NFUNC-001

**优先级**：Must Have（PM 提升，设计方向已明确）

**故事点**：待开发团队评估

**验收标准：**

- **AC-02-01** 背景与氛围
  - Given 业主进入舰桥页面
  - When 页面渲染完成
  - Then 页面背景为深邃宇宙深空（点缀星云与星光），叠加 80rpx 网格（青线 opacity 0.06），HUD 扫描线自上而下周期移动（5s 周期），舰体外部有冷蓝色引擎辉光

- **AC-02-02** 战舰透视布局
  - Given 舰桥页面完全渲染
  - When 查看页面整体布局
  - Then 页面自上而下依次呈现：舰首标识区（FREEARK 铭牌）→ 子系统隔舱 dock 区（4 个功能舱区：新风/能耗/水力/空气品质）→ 动力脊线（横向能量流动画，2.8s 周期）→ 房间隔舱网格区（客厅/主卧/次卧/书房/儿童房，全息文字标签悬浮）→ 舰尾引擎区（双引擎脉冲发光，1.8s 周期）

- **AC-02-03** 赛博朋克视觉一致性
  - Given 舰桥页面展示
  - Then 整体色调以深蓝、暗紫、霓虹青为主；功能舱区用发光轮廓线标注，正常运行青蓝色霓虹线、预警状态橙红色闪烁线、待机状态暗紫色弱光线；房间边缘发光线条代表环境系统运行状态；隔舱使用 clip-path polygon 异形造型

- **AC-02-04** TabBar 协调
  - Given 舰桥页面展示
  - Then ArkTabBar 底栏正常显示，"舰桥" tab 为 active 态（青色发光 pill），与页面视觉主题协调

- **AC-02-05** 无 Material Design 残留
  - Given 舰桥页面展示
  - Then 不存在白色卡片、蓝色头、灰色底等 Material Design 风格残留

## US-03 点击隔舱查看故障详情

**作为** 业主
**我希望** 点击任意子系统或房间隔舱时能看到具体的故障/预警列表
**以便** 了解具体什么设备出了什么问题

**关联需求**：REQ-FUNC-006、REQ-FUNC-007

**优先级**：Should Have

**故事点**：待开发团队评估

**备注**：PM 已确认 I-02 方案 A——半屏弹出抽屉（赛博朋克风格）。

**验收标准：**

- **AC-03-01** 子系统故障列表
  - Given 业主点击"新风"子系统隔舱（该隔舱标注有 3 条活跃故障）
  - When 点击后
  - Then 从底部弹出半屏抽屉（深色半透明背景+青色发光边框），展示该品类下全部活跃 FaultEvent 记录，每条含：设备名、故障类型、严重程度标签（"告警"/"预警"）、故障描述、首次发现时间；列表项带状态色左边框（红=故障/黄=预警）

- **AC-03-02** 房间故障列表
  - Given 业主点击"主卧"房间隔舱（该房间存在 1 条故障 + 1 条预警）
  - When 点击后
  - Then 半屏抽屉展示该房间内全部活跃故障/预警列表，结露预警事件一并列入（如有），每条标注所属设备和严重程度

- **AC-03-03** 正常隔舱点击
  - Given 业主点击一个无任何故障或预警的隔舱（如正常状态的"空气品质"隔舱）
  - When 点击后
  - Then 半屏抽屉展示提示"该子系统运行正常"或"该房间设备运行正常"

- **AC-03-04** 抽屉关闭
  - Given 故障/预警列表抽屉已打开
  - When 用户下拉关闭手柄、点击遮罩或返回手势
  - Then 抽屉关闭，返回舰桥透视图主页面，隔舱状态不变

## US-04 舰桥数据自动刷新与手动刷新

**作为** 业主
**我希望** 舰桥页面能自动更新故障状态，并且我也可以手动下拉刷新
**以便** 始终看到最新的系统状态，不会遗漏新产生的故障

**关联需求**：REQ-FUNC-009、REQ-FUNC-011

**优先级**：Must Have

**故事点**：待开发团队评估

**验收标准：**

- **AC-04-01**
  - Given 业主已停留在舰桥页面 30 秒以上
  - When 后台有新故障产生
  - Then 页面在下一个 30 秒轮询周期后自动更新隔舱颜色和故障计数（颜色从正常绿变为告警红、计数更新），不重建整个页面 DOM

- **AC-04-02**
  - Given 业主已停留在舰桥页面
  - When 某个活跃故障在轮询间隔内被恢复（`is_active` 变为 false）
  - Then 页面在下一个轮询周期后自动更新：对应隔舱颜色从红/黄恢复为正常青/绿、计数减少

- **AC-04-03**
  - Given 用户在舰桥页面
  - When 执行下拉手势
  - Then 触发手动刷新，页面显示刷新指示器（如扫描线加速动画），重新请求全部 API，更新所有隔舱状态

- **AC-04-04**
  - Given 舰桥页面在后台（用户切换到聊天 tab 或小程序进入后台）
  - When 页面不可见
  - Then 停止所有 API 轮询和 CSS 动画；页面重新可见时恢复轮询和动画

- **AC-04-05**
  - Given 单次 API 轮询请求失败（如网络超时）
  - When 该请求失败
  - Then 该 API 对应的隔舱保持上次成功获取的数据和颜色，其他 API 对应的隔舱仍正常更新；不因一个 API 失败导致全页锁定

## US-05 业主多座舱切换

**作为** 绑定了多个专有部分的业主
**我希望** 能在舰桥页面快速在不同座舱之间切换
**以便** 方便地巡视我名下所有座舱的设备状态

**关联需求**：REQ-FUNC-006、REQ-FUNC-010

**优先级**：Should Have

**故事点**：待开发团队评估

**验收标准：**

- **AC-05-01**
  - Given 业主已绑定 3 个 specific_part：A（3-1-702）、B（5-2-1503）、C（6-1-201），当前在座舱 A 的舰桥页面
  - When 点击座舱切换 picker，在下拉列表中选择座舱 B
  - Then 页面切换至座舱 B：房间隔舱更新为 B 的房间（如 B 有 4 个房间则展示 4 个隔舱）、子系统数据切换为 B 对应数据、健康指示器基于 B 的状态重新计算

- **AC-05-02**
  - Given 业主在座舱 A 的舰桥页面，然后切换到座舱 B
  - When 切换完成后
  - Then 页面标题区和 picker 显示座舱 B 的 location_name 或 specific_part

- **AC-05-03**
  - Given 业主仅绑定 1 个 specific_part
  - When 进入舰桥页面
  - Then 不显示座舱切换 picker（无切换必要），直接展示该座舱数据

- **AC-05-04**
  - Given 业主在座舱切换后
  - When 30 秒自动刷新触发
  - Then 仅刷新当前选中座舱的数据，不预加载其他座舱数据

## US-06 容错降级——系统 API 异常时不白屏

**作为** 业主
**我希望** 即使后端某个 API 暂时不可用，舰桥页面仍能部分展示
**以便** 不会因为一个接口故障而完全无法使用舰桥功能

**关联需求**：REQ-FUNC-012、REQ-NFUNC-005

**优先级**：Must Have

**故事点**：待开发团队评估

**验收标准：**

- **AC-06-01**
  - Given `/api/dashboard/device-fault-summary/` 接口返回 500 错误
  - When 业主进入或刷新舰桥页面
  - Then 子系统隔舱区域显示"数据不可用"标记（非空白），房间隔舱和整体健康指示器仍正常展示（因为它们依赖其他 API）；页面不白屏

- **AC-06-02**
  - Given 全部 4 个 dashboard API 均返回错误
  - When 业主进入舰桥页面
  - Then 页面展示赛博朋克风格错误横幅（如黄色左边框 + "数据暂时不可用，请下拉刷新重试"文字），战舰透视图隔舱结构保留（即便无数据填充），隔舱显示占位/离线标记

- **AC-06-03**
  - Given owner 的骨架接口（`/api/miniapp/owner/structure/`）已成功返回并缓存
  - When 后续实时参数接口（`/api/miniapp/owner/realtime-params/`）失败
  - Then 页面仍使用缓存的骨架数据渲染隔舱结构（房间名和布局不变），隔舱标注"实时数据暂不可用"

- **AC-06-04**
  - Given 页面展示错误横幅
  - When 用户手动下拉刷新后 API 恢复正常
  - Then 错误横幅消失，隔舱数据正常填充，颜色和计数更新为最新值
