## 角色定义

- **运维用户**：普通登录用户（`userInfo.role !== 'admin'`），具备查看、问答权限，无写操作权限。
- **管理员**：`userInfo.role === 'admin'` 的用户，具备运维用户的全部权限，外加工单审批权限。

### US-01 账号登录与会话管理

**作为**运维用户或管理员，
**我希望**通过账号密码登录微信小程序并保持登录状态，
**以便**在移动端随时访问 FreeArk 的监控和运维数据，无需每次重新登录。

#### AC-01.1 未登录时强制跳转登录页

```
Given 用户未登录（本地 Storage 中无 userToken）
When  用户打开小程序或访问任意需要鉴权的页面
Then  小程序自动跳转至登录页面
And   显示账号密码输入表单，无其他导航入口
```

#### AC-01.2 账号密码正确时登录成功

```
Given 用户在登录页输入正确的账号和密码
When  用户点击「登录」按钮
Then  小程序调用 POST /api/auth/login/
And   收到 HTTP 200 响应，响应体包含 token 和 user_info（含 role 字段）
And   token 与 user_info 持久化到本地 Storage（wx.setStorageSync）
And   小程序跳转至首页综合看板
And   若 role === 'admin'，用户可见管理员专属功能入口
```

#### AC-01.3 账号或密码错误时提示失败

```
Given 用户在登录页输入错误的账号或密码
When  用户点击「登录」按钮
Then  后端返回 HTTP 401
And   小程序弹出错误提示（wx.showToast）："账号或密码错误"
And   停留在登录页，输入框保留用户名，清空密码
```

#### AC-01.4 已登录时直接进入首页

```
Given 用户已登录（本地 Storage 中存有有效 userToken）
When  用户重新打开小程序
Then  小程序跳过登录页直接进入首页综合看板
```

#### AC-01.5 登出后清除本地会话

```
Given 用户已登录，当前在个人中心页面
When  用户点击「退出登录」
Then  弹出二次确认弹窗："确定退出登录？"
And   用户点击确认后，清除本地 Storage 中的 userToken 和 userInfo
And   跳转至登录页
```

### US-02 首页综合看板

**作为**运维用户，
**我希望**打开小程序后立即看到系统关键运行指标概览，
**以便**快速了解当前三恒系统的整体状态，发现异常时迅速导航至详情页。

#### AC-02.1 看板展示四个关键指标卡片

```
Given 用户已登录并进入首页
When  页面数据加载完成
Then  页面展示以下四张指标卡片：
      - 在线 PLC 数（格式：N / 总数）
      - 当前活跃故障数（N 条）
      - 当前结露预警数（N 条）
      - 今日总能耗（N kWh）
And   卡片内数字与后端实际数据一致
```

#### AC-02.2 异常状态卡片视觉突出

```
Given 当前存在活跃故障（故障数 > 0）或离线 PLC（在线 PLC < 总数）
When  首页加载完成
Then  对应指标卡片使用警告色（红色或橙色）高亮显示数字
And   状态正常时（故障数 = 0，PLC 全部在线）使用默认色展示
```

#### AC-02.3 快捷入口导航正确

```
Given 用户在首页
When  用户点击「故障」快捷卡片
Then  跳转至故障管理页（US-07）
When  用户点击「结露预警」快捷卡片
Then  跳转至结露预警页（US-08）
When  用户点击「AI 问答」快捷卡片
Then  跳转至 AI 问答页（US-11）
When  用户点击「设备监控」快捷卡片
Then  跳转至设备卡片列表页（US-04）
```

#### AC-02.4 下拉刷新更新数据

```
Given 用户在首页，当前显示的是已缓存数据
When  用户下拉页面触发刷新（onPullDownRefresh）
Then  小程序重新调用后端接口获取最新数据
And   刷新动画结束后，看板卡片显示最新数值
And   刷新过程中不阻塞用户与页面的交互
```

```
Given 用户在 PLC 状态页面

```
Given 用户在参数历史页面，当前显示「近 24 小时」数据

Then  小程序重新调用历史数据接口（传入对应时间范围参数）
And   图表更新为对应时间范围的数据
And   切换期间显示 loading 状态，不展示旧数据

**我希望**在移动端查看当前和历史故障列表，并支持状态/房间筛选，

```
Given 用户进入故障管理页

      故障描述、发生时间、所在房间、状态（未解决/已解决）

#### AC-07.2 按状态筛选故障

```
Given 用户在故障管理页

Then  列表只显示 status=未解决 的故障
When  用户选择「全部」

```
Given 系统中存在多个房间的故障记录

```

#### AC-07.4 下拉加载更多（分页）

```
Given 故障记录超过一页（默认每页 20 条）

#### AC-07.5 故障列表无写操作入口

```
Given 用户（无论是否管理员）在故障管理页
When  用户浏览故障列表
Then  列表中不出现任何「处理」「确认」「关闭」等写操作按钮

```

### US-11 AI 问答（方舟智能体 + 三恒专家）

**作为**运维用户，
**我希望**在移动端与 AI 专家对话，查询故障原因、操作规程或系统知识，
**以便**在现场快速获得技术支持，不需要打开桌面电脑。

#### AC-11.1 展示会话历史列表

```
Given 用户进入 AI 问答页
When  数据加载完成
Then  展示该用户的历史会话列表（最近 20 条），每条显示：
      会话摘要（首条消息前 30 字）、最后消息时间、使用的专家类型
And   按最后活跃时间降序排列
```

#### AC-11.2 创建新会话并发送消息

```
Given 用户在 AI 问答页
When  用户点击「新建会话」并选择专家类型（方舟智能体 / 三恒专家）
Then  进入空白聊天界面，标题显示所选专家名称
When  用户输入消息并点击发送
Then  消息出现在聊天气泡区（用户侧）
And   小程序通过 WebSocket 连接后端发送消息
And   AI 回复以流式方式逐步显示在聊天气泡区（AI 侧）
```

#### AC-11.3 流式消息正确渲染 Markdown

```
Given AI 正在以流式方式回复（包含 Markdown 格式：标题/列表/粗体/代码块）
When  流式内容不断追加
Then  消息气泡逐步展示内容，最终完整渲染 Markdown 格式
And   代码块有明显背景色区分，标题层级清晰
And   渲染过程中无明显卡顿（逐 chunk 更新间隔 ≤ 200ms）
```

#### AC-11.4 进入已有会话查看历史消息

```
Given 用户在会话列表页点击某个已有会话
When  页面加载完成
Then  展示该会话的完整历史消息（时间升序）
And   用户侧消息与 AI 侧消息气泡样式区分明显
And   自动滚动至最新消息
```

#### AC-11.5 WebSocket 断线后给出提示

```
Given 用户正在聊天，网络中断导致 WebSocket 断开
When  WebSocket 连接断开（onclose 事件触发）
Then  聊天页顶部显示「连接已断开，请检查网络后重试」提示条
And   发送按钮变为不可点击状态
And   用户可手动点击「重新连接」按钮触发 WebSocket 重连
```

### US-12 巡检工单查看与管理员审批

**作为**运维用户，
**我希望**在移动端查看巡检工单列表，了解当前待处理的工单状态；
**作为**管理员，
**我还希望**能在移动端直接审批（批准/拒绝）工单中的 PLC 写提案，
**以便**在现场巡检结束后不需要返回桌面端即可完成审批闭环。

#### AC-12.1 所有用户可查看工单列表

```
Given 用户（任意角色）进入巡检工单页
When  数据加载完成
Then  以列表形式展示工单，每条显示：
      工单编号、来源（自治/按需）、创建时间、状态（待审批/已批准/已拒绝/已执行）
And   支持按状态过滤（待审批/已完成/全部）
```

#### AC-12.2 管理员可见审批按钮

```
Given 管理员用户（role === 'admin'）进入巡检工单页
And   存在状态为「待审批」的工单
When  页面加载完成
Then  待审批工单的操作列中显示「批准」和「拒绝」按钮
And   非管理员用户的工单列表中不显示这两个按钮
```

#### AC-12.3 管理员批准工单

```
Given 管理员在工单列表看到一条「待审批」工单
When  管理员点击「批准」按钮
Then  弹出二次确认弹窗："确定批准该 PLC 写操作工单？"
When  管理员点击「确定」
Then  小程序调用工单审批 API（APPROVE）
And   后端返回成功后，该工单状态更新为「已批准」
And   列表中该工单状态标签同步更新，审批按钮消失
```

#### AC-12.4 管理员拒绝工单

```
Given 管理员在工单列表看到一条「待审批」工单
When  管理员点击「拒绝」按钮
Then  弹出弹窗，要求输入拒绝原因（可选填）
When  管理员确认拒绝
Then  小程序调用工单审批 API（REJECT）
And   后端返回成功后，该工单状态更新为「已拒绝」
And   列表同步更新
```

### US-13 个人中心 — 修改密码

**作为**任意已登录用户，
**我希望**在移动端修改自己的登录密码，
**以便**定期更新密码保障账号安全。

#### AC-13.1 修改密码成功

```
Given 用户进入个人中心 — 修改密码页面
When  用户正确填写旧密码、新密码（≥6位）、确认新密码，并点击「确认修改」
Then  小程序调用后端改密接口
And   后端返回成功后，显示「密码修改成功，请重新登录」提示
And   清除本地 Storage 中的 token 和 userInfo
And   跳转至登录页
```

#### AC-13.2 旧密码错误时提示

```
Given 用户输入了错误的旧密码，新密码格式合法
When  用户点击「确认修改」
Then  后端返回错误响应（HTTP 400/401）
And   小程序显示错误提示："原密码不正确"
And   停留在改密页面，清空所有密码输入框
```

#### AC-13.3 新密码与确认密码不一致时前端拦截

```
Given 用户填写新密码与确认新密码不一致
When  用户点击「确认修改」
