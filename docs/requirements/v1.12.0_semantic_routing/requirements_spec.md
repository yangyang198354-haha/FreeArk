## 0. 问题陈述

### 0.1 背景现状（代码锚定）

当前路由管线（`api/langgraph_chat/router.py` + `orchestrator._route`，P0/P1 系列已上线）：

1. **关键词短路**（P0-1）：当前问题唯一无撞车命中 `ROUTE_KEYWORDS` → 直接定专家、跳过 LLM（0s）。
2. **LLM 分类器**（阶段 D）：`classify_experts_llm` 调 router_llm（deepseek-v4-flash，temp 0），Pi 实测约 **2.5s/次**。
3. **兜底链**：关键词 → 粘性（P0-2）→ 域外 `[]`（P1-2）→ DEFAULT。

其中关键词路由是确定性主力 + LLM 兜底前的短路层。

### 0.2 痛点（评测集实证，见 routing_eval/dataset.jsonl）

1. **关键词覆盖脆弱**：同义/改写词命中不到。评测实例：
   - `know-010`「这套系统平时是怎么除湿和换气的？」——知识问题但无任一关键词 → 离线落 DEFAULT energy（错），只有 LLM（2.5s）能救。
   - `insp-006`「最近设备有没有什么异常情况」——"异常"非关键词 → 同上。
2. **词表只增不减、靠人肉维护、易撞车**：`router.py` 注释多处自述"不收某词防误路由"（如刻意不收"控制/调节"）、`ener-001`「在线率」含"在线"撞 inspection。每加一类表达就要改词表 + 防撞车，长期不可维护。
3. **LLM 兜底成本**：所有关键词够不着的单意图问题都付 ~2.5s LLM 往返。P0-1 短路当前覆盖 71%（27/38 旧基线），剩余靠 LLM——其中相当部分是"语义清晰但无关键词"的问题，本可更快更准地判定。

### 0.3 本版本目标

用**语义相似度路由**（复用 v1.4.0 已有的 bge-m3 embedding 基建，见 `api/rag_service.py`）作为**关键词短路与 LLM 分类器之间的"高置信中间层"**：
- 把短路能力从"字面关键词命中"扩展到"语义贴近某专家"，覆盖同义/改写/无关键词的单意图问题；
- 在高置信时跳过 LLM（潜在延迟收益，**取决于 Phase-0 实测的 embedding 延迟**）；
- 降低关键词表维护负担（语义层吸收长尾表达，词表可瘦身或冻结）；
- **严格不回归**：低置信仍交 LLM；复合/域外仍由现有路径处理；全程 fail-open（embedding 故障退回今天的管线）；一键开关回退。

> ⚠️ **核心未决问题（决定立项价值）**：语义路由是否为**延迟收益**取决于 Pi 上 embedding 查询的真实延迟。bge-m3 经 SiliconFlow（OpenAI 风格 /embeddings），短文本通常数百 ms，但 Pi WiFi 偶发劣化（`rag_service` 注释：查询端 12s 超时、2 次重试）。**若 embedding 稳定显著 < 2.5s → 既提覆盖又降延迟；若与 LLM 相当或更慢 → 仅提覆盖/可维护性，需重新权衡是否值得**。故 Phase-0 PoC 实测为立项门控（见 §5）。

## 1. 范围与边界

### 1.1 纳入范围

| 编号 | 功能域 | 摘要 |
|------|--------|------|
| REQ-FUNC-SR-01 | 专家范例库 | 每个专家维护一组代表性范例问句（exemplars），来源见 OQ-SR-01 |
| REQ-FUNC-SR-02 | 范例向量化与缓存 | 启动/刷新时对范例批量 embed，进程内缓存（仿 RagVectorCache） |
| REQ-FUNC-SR-03 | 查询语义打分 | 路由时 embed 当前问题，对各专家范例集算余弦，得每专家相似度 |
| REQ-FUNC-SR-04 | 高置信判定 | top 专家分数 ≥ τ 且与次高 margin ≥ δ → 语义短路定该单专家 |
| REQ-FUNC-SR-05 | 管线集成 | `_route` 在关键词短路之后、LLM 之前插入语义层；低置信穿透到 LLM |
| REQ-FUNC-SR-06 | fail-open | embedding 不可达/超时/异常 → 静默退回 LLM 分类器（今天的行为），不打挂聊天 |
| REQ-FUNC-SR-07 | 开关 | `LANGGRAPH_ROUTER_SEMANTIC`（env，默认见 OQ-SR-05）；置 False 完全回退现状 |
| REQ-FUNC-SR-08 | 阈值标定 | τ/δ 由评测集标定（留一法），写入 settings，可调 |
| REQ-FUNC-SR-09 | 评测 harness 扩展 | routing_eval 增"语义模式"：embed 数据集，留一法评准确率 + 与离线/live 对比 |
| REQ-FUNC-SR-10 | 可观测 | 语义短路命中/穿透/fail-open 计数与分数 log（便于标定与排障） |

### 1.2 不纳入范围

- 引入独立向量库（Chroma/Qdrant 等）——复用现有 MySQL/内存余弦基建，与 v1.4.0 一致。
- Docker——禁止（物理机约束 [[project_infrastructure]]）。
- 更换 embedding 模型/供应商——复用现网 bge-m3 / SiliconFlow（`RAG_EMBEDDING_*`）。
- 复合意图的语义多选——本期语义层只做"高置信单专家短路"，复合仍交 LLM（margin 低自然穿透）。
- 域外语义判定——OOD 仍由 P1-2 的 LLM `[]` 信号处理（语义层不负责判 OOD）。
- 退役关键词路由——本期语义层**叠加**在关键词之上，关键词短路/兜底保留；词表瘦身留待后续。

## 2. 用户故事与验收

**US-SR-01**（无关键词的单意图问题被快速正确路由）
- *作为* 用户，*当* 我问"这套系统平时怎么除湿换气"（无关键词的知识问题），*那么* 系统应语义判定为 sanheng-knowledge 并（在高置信时）跳过 LLM。
- AC-SR-01-01：`know-010`、`insp-006` 等"无关键词单意图"用例，语义模式下精确命中 expected。
- AC-SR-01-02：命中时不调用 LLM 分类器（log 可证）。

**US-SR-02**（不回归既有正确路由）
- AC-SR-02-01：所有 `keyword_floor=true` 用例仍精确命中（关键词短路在语义层之前，不受影响）。
- AC-SR-02-02：P0-2 粘性、P1-2 域外、写确认门、护栏相关测试全绿。
- AC-SR-02-03：复合用例（`comp-*`）不被语义层误短路为单专家（margin 门控）。

**US-SR-03**（故障不打挂）
- AC-SR-03-01：embedding API 超时/不可达时，路由静默退回 LLM 分类器，聊天正常。
- AC-SR-03-02：`LANGGRAPH_ROUTER_SEMANTIC=False` 时行为与 v1.12.0 前逐字节一致。

**US-SR-04**（可标定可观测）
- AC-SR-04-01：harness 语义模式输出留一法准确率 + 每类别分数分布，支撑 τ/δ 选择。
- AC-SR-04-02：生产 log 记录语义命中率、穿透率、fail-open 率。

## 5. Phase-0 PoC 门控（立项前必做）

在 Pi 上用真实 bge-m3 端点测量，**结果决定是否继续 full build 及 OQ-SR-06**：

1. **embedding 查询延迟分布**：对评测集 42 条各 embed 一次，记录 p50/p90/p99 与超时率（多跑几轮覆盖 WiFi 抖动）。
2. **语义路由准确率（留一法）**：对每条用例，用其余用例作各专家范例，embed 本条 → 取各专家最大余弦 → argmax 是否 == expected 单专家；扫 τ/δ 网格出准确率曲线。
3. **与现有对比**：语义 vs 离线关键词地板（78.6%）vs live LLM（86.8%），特别看 `know-010`/`insp-006` 等关键词盲点是否被语义救回。
4. **判定**：
   - p90 延迟 < 1.5s 且语义准确率 ≥ live LLM 附近 → 推进 full build，OQ-SR-06=默认收益路径。
   - 延迟高但准确率好 → OQ-SR-06=A（覆盖增强为主），重审延迟预期。
   - 准确率不及关键词地板 → 暂缓，回炉范例/阈值。

PoC 只读、零生产改动（独立脚本，仿 `scripts/analysis/router_probe.py`）。

## 6. 风险

| 编号 | 风险 | 缓解 |
|------|------|------|
| RISK-SR-01 | embedding 端点延迟/超时使热路径变慢 | 有界超时 + fail-open 退 LLM；Phase-0 实测门控；默认关 + 灰度 |
| RISK-SR-02 | 范例质量差致语义误路由 | 范例源自评测集（真实+事故）；高 τ + margin 双门；低置信穿透 LLM；不变式测试防回归 |
| RISK-SR-03 | 语义把复合意图误短路成单专家 | margin 门（两专家都高分→margin 小→穿透 LLM）；comp-* 用例纳入回归 |
| RISK-SR-04 | 关键词盲点同时也是语义盲点（领域词偏） | 评测集持续扩充；语义+关键词+LLM 三层互补，任一层兜底 |
| RISK-SR-05 | 启动时范例 embedding 失败致语义层空转 | fail-open（无范例→直接穿透 LLM）；可选 OQ-SR-02=B 持久化 |
