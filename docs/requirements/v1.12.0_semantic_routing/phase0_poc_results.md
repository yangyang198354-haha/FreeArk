## 2. 留一法语义准确率

- **纯 argmax（无阈值）：30/35 = 85.7%**，与 live LLM(86.8%)同档——语义本身分得开。
- **τ/δ 网格**：存在干净的**零误路由操作点 τ=0.65, δ=0**：短路 18/35（覆盖 51.4%），命中精度 **100%**，且这 18 条 margin 均 ≥0.10（高分高距，很稳）。
- 低 τ 提覆盖但引入误判（τ=0.30 → 覆盖 100% 但精度降到 85.7%）。

## 3. 关键词盲点（keyword_floor=false 单专家）是否被语义救回

| 用例 | 语义判定 | 分数/margin | 结果 |
|------|----------|-------------|------|
| `insp-006`「异常情况」 | inspection | 0.77 / 0.25 | ✓ 高置信救回（τ=0.65 命中） |
| `ener-002`「实时温湿度」 | energy | 0.72 / 0.30 | ✓ 高置信救回（τ=0.65 命中） |
| `know-010`「除湿换气」 | sanheng | 0.54 / 0.10 | ✓ argmax 对，但 <0.65 → 穿透 LLM（不回归，无提速） |
| `stick-001/002/003` 追问 | 各对 | margin 0.01~0.03 | ✓ argmax 对，但低 margin → 穿透 LLM（粘性/LLM 接） |
| `ener-001`「总能耗和在线率」 | inspection | 0.61 / 0.06 | ✗ argmax 错（本就 debatable-label 复合性问题）；低分 → 穿透 LLM |

语义混淆点（argmax 错 5 条：know-006/ener-001/insp-002/insp-005/ctrl-002）**全部分数低或 margin 小**，
在 τ=0.65 操作点下均被拒绝短路 → 穿透 LLM，**不会造成误路由**。这正是双阈值门控的价值。

## 4. 判定：GO（保守推进），但收益定位 = 覆盖/可维护性为主、延迟为辅

按 §5 判定标准：p90<1.5s ✓、存在 100% 精度操作点 ✓、救回部分关键词盲点 ✓ → **GO**。但须诚实定位：

- **延迟收益 modest**（~1.1s/条，仅作用于语义短路命中的子集），非 P0-1 那种 0 延迟。
- **净增量覆盖有限**：τ=0.65 时 18 条命中里多数是 keyword_floor=true（关键词短路已在语义层前处理，
  语义根本看不到）。语义在生产的**真正新增**= 关键词够不着且语义≥0.65 的（实测 insp-006/ener-002 类 + 未来长尾改写）。
- **主要价值 = 关键词表可冻结/瘦身**（语义吸收长尾表达）+ 高置信盲点提速。对应 OQ-SR-06=（A 覆盖增强为主）。

**建议 τ/δ 初值**：τ=0.65, δ=0.05（保守，零误路由；随评测集增长再标定）。
**建议默认态**：OQ-SR-05=B（默认关，Phase-1 上线代码零行为变化，灰度验证后再开）。

> 注：留一法仅 35 条小样本，100% 精度偏乐观。生产取保守 τ + fail-open + 默认关 + 灰度，风险可控。
> 若评估认为"覆盖增强为主、延迟收益有限"不值新增热路径远端依赖的复杂度，亦可据此**不推进** full build——
> 这是 PoC 暴露的真实权衡，交用户/评审定夺。
